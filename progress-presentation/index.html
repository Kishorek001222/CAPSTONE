2. Components

Frontend (React)

  • Role-based dashboards for Student, University, Employer.
  • Credential issue/verify UI and workflow views.
  • Main and Demo mode separation with mode banner.

Backend (Node/Express)

  • Auth, credential APIs, role-based access control.
  • Credential metadata persistence and orchestration.
  • Integration readiness and health visibility.

Blockchain (Ethereum)

  • Smart contract for DID and credential status state.
  • Credential issue/revoke/verify primitives.
  • Read-only verification path via RPC fallback.

Storage (MongoDB + IPFS)

  • MongoDB for users and credential metadata.
  • IPFS (Pinata) for verifiable credential documents.
  • Hybrid model: on-chain proof + off-chain metadata.

3. Design and Architecture

->
Express API
Ethereum Contract
<->
IPFS + MongoDB

Layered Design Decisions

  • Verification trust anchored on-chain to avoid central dependency.
  • Credential documents stored off-chain to reduce gas and improve scalability.
  • Backend orchestration for consistency, validation, and operational control.
  • Separation of concerns between UI, API orchestration, and chain logic.

Operational Flow

  1. University creates credential payload.
  2. Payload is normalized and hashed deterministically.
  3. Metadata pinned to IPFS and CID captured.
  4. Credential state issued and later verified from blockchain.
